# Developer Interview Take-Home Task

## Task:

Design and implement a CRUD (Create, Read, Update, Delete) application using Java or Kotlin that manages invoices and their corresponding line items. The application should:

- **Create:** Allow users to generate new invoices and add multiple line items to each invoice. Each line item should include details like item description and price.
- **Read:** Enable users to view and search existing invoices, along with their associated line items.
- **Update:** Support modification of existing invoices and their line items. Allow users to edit invoice details, add or remove line items, and update line item information.
- **Delete:** Implement functionality to delete invoices and their corresponding line items.

The application should also allow users to pay for invoices and credit the paid amount. This feature should include the following functionalities:

- **Pay Invoice:** Users can initiate a payment for an invoice.
- **Pay Invoice:** Users can initiate a partial payment for an invoice.
- **Refund Paid Amount:** User can request a refund.

Consider using appropriate data structures and algorithms to represent invoices and line items, and choose a suitable database or storage mechanism to persist the data. Implement a user interface or API to interact with the application and perform CRUD operations.

## Additional Notes:

- Applicants are free to use any Java or Kotlin framework or library of their choice.
- Applicants are encouraged to use a version control system such as Git to track their changes.

**Good luck!**
